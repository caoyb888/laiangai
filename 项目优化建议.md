## 三、改进建议

### 3.1 建立后端接口完成标准（Definition of Done）

每个接口在提测前必须满足：

```
□ 所有 HTTP 方法均已实现（含 DELETE/PATCH）
□ BackgroundTask 使用的数据在启动前已 commit
□ 异步路由中无同步阻塞调用（CPU 密集用 asyncio.to_thread）
□ 返回给前端的 URL 不含内网主机名
□ 接口路径已在接口文档（或 OpenAPI）中登记
```

将此清单加入 PR 模板，Code Review 时逐项核对。

---

### 3.2 前后端接口路径在开工前对齐

**现状问题：** 前端写死 `/compare/`，后端实际是 `/compare/tasks`，联调时才发现 404。

**改进措施：**
1. 以后端 FastAPI 自动生成的 `/docs`（Swagger UI）为唯一接口文档
2. 前端在开始联调前，先访问 `http://localhost:8000/docs` 确认路径
3. 前端 API 调用集中在 `src/api/` 目录，不在组件里散落路径字符串

---

### 3.3 环境依赖写进 Dockerfile，不依赖宿主机隐性安装

本次中文字体、BGE-M3 模型都是"运行时才发现缺失"。

**改进措施：**

```dockerfile
# 在 backend/Dockerfile 中明确声明字体依赖
RUN apt-get install -y fonts-noto-cjk

# 在 docker-compose.yml 的 environment 中声明模型路径
# 在 README 或 docs/部署手册.md 中列出所有外部文件的下载步骤
```

**模型文件下载规范：**
- 禁止 `wget -c`（断点续传会污染 LFS 指针文件）
- 统一使用 Python `requests` 流式下载，或 `huggingface-cli download`
- 下载后校验 SHA256，写入 `infra/checksums.txt`

---

### 3.4 前端布局组件使用检查清单

本次出现了 flex 布局、虚拟滚动两类典型问题。

**flex 布局规范：**
```css
/* 父容器是 display:flex 时，子容器必须明确声明宽度策略 */
.child-fill  { flex: 1; min-width: 0; }   /* 占满剩余空间 */
.child-fixed { flex-shrink: 0; width: Xpx; } /* 固定宽度 */
```

**虚拟滚动选型规范：**
| 场景 | 组件 |
|------|------|
| 每行高度固定 | `RecycleScroller`（性能最优） |
| 每行高度可变（文本内容） | `DynamicScroller` + `DynamicScrollerItem` |

---

### 3.5 MinIO 文件访问统一走后端代理

CLAUDE.md §4.4 已经写明"禁止前端直接访问 MinIO"，但实现时用了预签名 URL 直接返回给前端，违反了这条规范。

**落地方式：**
```
前端 → GET /api/v1/xxx/download/{id}（带 Authorization）
后端 → 从 MinIO 取文件 → StreamingResponse 返回给前端
```

永远不要把含有内网主机名的 URL 出现在 API 响应里。

---

### 3.6 增加冒烟测试脚本，覆盖核心链路

本次所有问题都是在人工点击时才发现。建议在 `tests/smoke/` 下维护一个端到端脚本：

```python
# tests/smoke/test_e2e.py
# 1. 登录，拿 token
# 2. 上传文档，轮询直到 parse_status=done
# 3. 创建比对任务，轮询直到 status=done，断言 total_diffs > 0
# 4. 导出 PDF，断言响应 Content-Type=application/pdf，文件大小 > 1KB
```

每次部署后执行，5 分钟内发现主流程问题。

---

### 3.7 git 工作流改进

**本次发现的问题：**
- `colbert_linear.pt`、`sparse_linear.pt` 模型文件出现在项目根目录（应在 `.gitignore`）
- 远程有未同步的提交，push 时发生冲突

**改进措施：**

```gitignore
# .gitignore 补充
*.pt
*.bin
*.onnx
/models/
```

推送前养成习惯：
```bash
git fetch origin && git status   # 先确认本地与远程无分歧
git push origin <branch>
```
