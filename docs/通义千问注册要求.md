# 通义千问注册要求

> 任务12（LLM API 客户端）开发前，需在通义千问（阿里云 DashScope）完成以下准备工作。

---

## 一、账号与开通

1. **注册/登录阿里云账号** — [aliyun.com](https://www.aliyun.com)
2. **开通 DashScope 服务** — 进入「百炼」或「DashScope」控制台，完成服务开通（首次需实名认证）

---

## 二、获取 API Key

1. 进入 DashScope 控制台 → **API Key 管理**
2. 创建一个新的 API Key
3. **立即复制保存**（只显示一次）
4. 按 CLAUDE.md 规范写入本地 `.env` 文件：
   ```bash
   LLM_API_KEY_PRIMARY=sk-xxxxxxxxxxxxxxxxxxxxxxxx
   ```
   > `.env` 已在 `.gitignore` 中，禁止提交到版本库

---

## 三、确认可用模型

任务12使用的模型建议选择：

| 用途 | 推荐模型 |
|------|---------|
| 合同条款语义分析（主） | `qwen-long` 或 `qwen-max` |
| 低成本/快速分析 | `qwen-turbo` |

在控制台确认账号对目标模型有调用权限（部分模型需单独申请）。

---

## 四、额度与限流确认

- 查看当前**免费额度**或充值情况（Token 余额）
- 了解目标模型的 **RPM（每分钟请求数）** 和 **TPM（每分钟 Token 数）** 限制，这影响任务12中限流逻辑的参数设置

---

## 五、备用 API（DeepSeek）

按 CLAUDE.md §2.2，还需准备备用 Key：

1. 登录 [platform.deepseek.com](https://platform.deepseek.com)
2. 创建 API Key，写入 `.env`：
   ```bash
   LLM_API_KEY_BACKUP=sk-xxxxxxxxxxxxxxxxxxxxxxxx
   ```

---

## 六、本地验证（可选但推荐）

开发前用最简单的方式验证 Key 有效：

```bash
curl https://dashscope.aliyuncs.com/api/v1/services/aigc/text-generation/generation \
  -H "Authorization: Bearer $LLM_API_KEY_PRIMARY" \
  -H "Content-Type: application/json" \
  -d '{"model":"qwen-turbo","input":{"messages":[{"role":"user","content":"hello"}]}}'
```

收到正常响应后，即可开始任务12的开发。
